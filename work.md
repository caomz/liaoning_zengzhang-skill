# PART A：增长工作流详解

> 本文件为梁宁·增长思维30讲技能的 PART A 部分，详细阐述六大增长模块的工作流、方法论、工具、输出格式、经验规则、算法模型和陷阱反模式。

---

## 模块一：增长地图

### 目的

判断企业/产品/组织当前所处阶段，明确核心任务。

### 核心模型：企业定位模型

| 阶段 | 核心特征 | 关键问题 | 核心任务 |
|------|----------|----------|----------|
| 草莽 | 看到机会，快速冲！ | 谁能活下来？ | 找到破局点 |
| 腰部 | 站稳脚跟，有明确对手 | 如何差异化？ | 建立护城河 |
| 头部 | 行业前三，开始制定规则 | 如何持续领先？ | 持续正确决策 |
| 顶级 | 生态系统构建者 | 如何穿越周期？ | 投资创新 |

### 分析流程

```
1. 收集信号：市场规模、竞争格局、用户认知、团队能力
2. 阶段判断：对照四阶段特征，定位当前阶段
3. 核心任务：根据阶段确定首要任务
4. 风险识别：该阶段的典型死亡方式
5. 行动建议：该阶段最重要的一件事
```

### 输出格式

```
## 增长地图诊断

**当前阶段**：腰部企业

**阶段特征匹配**：
- [x] 有明确竞争对手
- [x] 用户认知相对稳定
- [ ] 差异化优势尚未建立 ⚠️

**核心任务**：建立差异化护城河

**典型风险**：被头部品牌下沉压制，或被草莽对手侧翼包抄

**立即行动**：
1. 找到细分市场中你能持续领先的那个点
2. 集中资源在这个点上建立相对优势
3. 避免分散精力同时追多个机会
```

### 经验规则

1. 阶段不可跳，只能加速通过
2. 每个阶段有每个阶段的"正确"，不是阶段错了，是你不在那个阶段
3. 草莽阶段想太多战略会死，核心是活下来
4. 腰部阶段是大多数企业的终局，因为差异化不够
5. 头部企业的核心能力是"持续作出正确决定"

## 模块二：机会识别

### 目的

找到可持续的增长切入点，即"破局点"。

### 核心模型组

#### 2.1 穷举发散法

**目的**：系统性地发现所有可能机会

**方法**：召开460件事工作坊
- 40人 × 11个维度 = 460件事
- 11个维度：产品、服务、渠道、技术、供应链、品牌、用户、组织、资本、合作、生态

**流程**：
```
1. 发散：列出所有可能的机会，不评判
2. 分类：按11个维度归类
3. 筛选：去掉明显不可行的
4. 评估：对剩余机会进行破局点评分
```

#### 2.2 破局点公式

**公式**：`破局点 = 相对广谱 × 高频 × 体验可控`

| 要素 | 含义 | 判断标准 |
|------|------|----------|
| 相对广谱 | 覆盖足够大的目标人群 | 不是小众，是相对广谱 |
| 高频 | 用户反复使用/购买 | 低频产品难以形成网络效应 |
| 体验可控 | 用户体验能超出预期 | 不能被供应商或外部因素破坏 |

**评估方法**：
- 对每个候选机会，三项都打分（1-5分）
- 破局点评分 = 三项乘积
- 得分>60分的才可能成为破局点

#### 2.3 爆品公式

**公式**：`爆品 = 供应链创新 × 爆发品类 × 新流量`

| 要素 | 含义 | 案例 |
|------|------|------|
| 供应链创新 | 结构性成本降低或效率提升 |Costco、必要商城 |
| 爆发品类 | 处于需求爆发期的品类 | 智能手机、新茶饮 |
| 新流量 | 新的用户聚集渠道 | 抖音、小红书 |

**三者缺一不可**。

#### 2.4 品牌护城河

**三层壁垒**：

| 层级 | 含义 | 案例 |
|------|------|------|
| 用户溢价 | 用户愿意为此付更高价格 | Apple |
| 商场折扣 | 渠道商主动推荐 | 宝洁系 |
| 用户自拍 | 用户主动传播 | 小红书种草 |

**评估**：你的品牌在哪个层级建立了壁垒？

#### 2.5 第二曲线

**概念**：企业增长会遭遇天花板，需要跨越非连续性找到新增长曲线

**时机**：在第一曲线还在增长时就开始探索第二曲线

**判断**：
- 核心业务是否触及天花板？
- 是否找到了新的技术/用户/市场机会？
- 是否有足够的资源和意志力跨越非连续性？

### 输出格式

```
## 机会诊断报告

**机会清单**（按破局点评分排序）：

| 机会 | 广谱性 | 高频性 | 可控性 | 总分 | 备注 |
|------|--------|--------|--------|------|------|
| 机会A | 4 | 5 | 4 | 80 | ⚡推荐 |
| 机会B | 3 | 4 | 3 | 36 | 备选 |

**核心建议**：
集中资源在机会A，集中火力找到破局点。

**爆品可能性**：
- 供应链创新：✓ 有潜力（某环节效率提升）
- 爆发品类：✓ 处于需求爆发期
- 新流量：⚠️ 需要找到新的流量获取方式

**品牌护城河规划**：
优先建立"用户自拍"层壁垒，通过口碑传播降低获客成本。
```

## 模块三：模式设计

### 目的

设计可持续的商业模式，确保增长有根。

### 核心模型：商业模式三要素

**公式**：`商业模式 = 目标 + 要素 + 连接`

| 要素 | 含义 | 问题 |
|------|------|------|
| 目标 | 解决什么人的什么问题 | 目标用户是否清晰？需求是否真实？ |
| 要素 | 需要哪些资源和能力 | 你的核心要素是什么？稀缺吗？ |
| 连接 | 要素如何组合起来 | 你的模式能持续运转吗？ |

### 核心模型：增强回路与调节回路

#### 增强回路

**概念**：系统中的正反馈循环，持续为系统增加能量

**案例**：
- 抖音：更多内容 → 更多用户 → 更多创作者 → 更多内容
- 美团：更多商家 → 更多用户 → 更多订单 → 更多商家

**设计检查**：
```
1. 你的业务有增强回路吗？
2. 增强回路的起点是什么？
3. 增强回路的终点是什么？（何时会停下来？）
```

#### 调节回路

**概念**：系统中的负反馈，防止系统过热

**核心洞察**：心理能量有尽头，靠模式不靠意志力

**设计原则**：
- 不要依赖意志力来维持系统运转
- 要设计自动运转的机制
- 调节回路让人停下来，喘息，回归正常

### 核心模型：用户增长飞轮

**公式**：`飞轮 = 核心交付 × 增强回路 × 网络效应`

| 要素 | 含义 |
|------|------|
| 核心交付 | 你给用户提供的核心价值是什么？ |
| 增强回路 | 如何让核心交付带来更多用户？ |
| 网络效应 | 如何让用户之间产生正向依赖？ |

### 四种商业模式

| 模式 | 轻/重 | 特征 | 案例 |
|------|------|------|------|
| 连接器模式 | 轻 | 去哪儿、OYO | 平台定位，撮合交易 |
| 整合服务模式 | 重 | 亚朵 | 重体验，深连接 |
| 流量平台模式 | 中 | 抖音 | 动态不平衡，网络效应，撮合 |
| 产业中台S2B2C | 中 | 必要商城 | 系统能力赋能小B |

### S2B2C深度解析

**概念**：S是大供应链平台，赋能众多小B，小B服务C端用户

**价值**：
- S端：系统能力（供应链、技术、运营）
- B端：离用户近，有情感连接，有本地化能力
- C端：获得更好的服务体验

**适合场景**：产业互联网，传统行业升级

### 输出格式

```
## 商业模式诊断

**三要素分析**：

| 要素 | 内容 | 评估 |
|------|------|------|
| 目标 | 解决的什么问题 | ⚠️ 目标用户模糊 |
| 要素 | 核心能力和资源 | ✓ 供应链有优势 |
| 连接 | 模式如何运转 | ⚠️ 增强回路不明显 |

**回路诊断**：

增强回路：存在，但强度不足
- 起点：核心交付
- 当前堵点：增强回路不够强

调节回路：缺失
- 建议：设计调节机制，防止系统过热

**飞轮评估**：
- 核心交付：✓ 清晰
- 增强回路：⚠️ 需要强化
- 网络效应：❌ 尚未建立

**模式建议**：从连接器模式向整合服务模式转型，增加用户粘性
```

## 模块四：组织诊断

### 目的

评估组织能力是否支撑增长，识别组织层面的增长瓶颈。

### 核心模型：组织五阶段

| 阶段 | 特征 | 核心能力 | 死亡方式 |
|------|------|----------|----------|
| 家庭 | 靠创始人能力 | 个人英雄主义 | 创始人离开 |
| 部落 | 有了小团队 | 执行力 | 团队内斗 |
| 村庄 | 有了部门和层级 | 组织能力 | 部门墙 |
| 城市 | 事业部/子公司 | 生态系统 | 大企业病 |
| 国家 | 多元化集团 | 资本运作 | 战略失焦 |

**核心洞察**：组织阶段不可跨越，只能加速通过。

### 核心模型：关系四阶段

**公式**：`关系深化 = 理想 → 冲突 → 整合 → 协同`

| 阶段 | 特征 | 任务 |
|------|------|------|
| 理想 | 你好我好大家好 | 保持清醒，不要被理想化蒙蔽 |
| 冲突 | 差异显现，矛盾产生 | 直面冲突，不要回避 |
| 整合 | 找到共同目标 | 建立真正的信任 |
| 协同 | 1+1>2 | 释放组织能量 |

### 核心模型：权力三指标

| 指标 | 含义 | 诊断问题 |
|------|------|----------|
| 信息流 | 谁知道什么 | 信息是否通畅？哪些人信息闭塞？ |
| 利益链 | 谁得到什么 | 利益分配是否清晰合理？ |
| 资源调动 | 谁能调什么 | 资源能否流向最重要的地方？ |

### 核心模型：字节跳动创新飞轮

**三要素**：`透明 × OKR × 飞书`

| 要素 | 作用 |
|------|------|
| 透明 | 信息平等，减少信息损耗 |
| OKR | 目标对齐，激发自驱 |
| 飞书 | 工具支撑，协作高效 |

### 输出格式

```
## 组织诊断报告

**组织阶段**：村庄阶段

**五阶段评估**：
- [x] 跨过了家庭阶段的创始人依赖
- [x] 建立了小团队执行力
- [ ] 部门协作存在墙 ⚠️
- [ ] 需要向城市阶段的生态系统演进

**关系四阶段**：
当前处于冲突阶段向整合阶段过渡
- 建议：明确共同目标，建立互信

**权力诊断**：

| 维度 | 状态 | 问题 |
|------|------|------|
| 信息流 | ⚠️ | 产品和技术的连接不畅 |
| 利益链 | ⚠️ | 股权激励覆盖不足 |
| 资源调动 | ✓ | 资源能流向核心项目 |

**字节跳动飞轮对标**：
- 透明：⚠️ 需要增加跨部门信息共享
- OKR：✓ 已建立，个别部门执行变形
- 飞书：❌ 尚未统一工具，协作效率低

**核心建议**：
1. 打通产品-技术信息流，建立定期沟通机制
2. 完善利益链，增加核心员工股权覆盖
3. 统一协作工具，提升协作效率
```

## 模块五：借势策略

### 目的

借助外部资源放大增长势能，实现指数级增长。

### 核心模型：战略杠杆

**公式**：`战略杠杆 = 支点 × 杠杆 × 指北针`

| 要素 | 含义 | 问题 |
|------|------|------|
| 支点 | 你的核心能力是什么 | 是真正的能力还是一厢情愿？ |
| 杠杆 | 什么能放大你的能力 | 资本？技术？政策？流量？ |
| 指北针 | 你要去哪 | 目标清晰吗？稳定吗？ |

**核心原则**：先找支点，再找杠杆，最后确定方向。

### 核心模型：利益相关人地图

**矩阵**：`利益相关人 = 影响力 × 利益`

| 类型 | 高影响力+高利益 | 高影响力+低利益 | 低影响力+高利益 | 低影响力+低利益 |
|------|----------------|----------------|----------------|----------------|
| 策略 | 重点管理 | 保持满足 | 保持信息 | 监控 |
| 案例 | 核心用户、关键供应商 | 监管机构 | 媒体 | 普通用户 |

### 核心模型：闪电式扩张

**阶段**：`海盗 → 海军陆战队 → 军队 → 警察`

| 阶段 | 特征 | 核心能力 | 死亡方式 |
|------|------|----------|----------|
| 海盗 | 找PMF | 产品力 | 产品不对 |
| 海军陆战队 | 快速扩张 | 执行力 | 管理失控 |
| 军队 | 规模化 | 组织能力 | 大企业病 |
| 警察 | 维护生态 | 生态治理 | 创新者窘境 |

**适用条件**：找到了真正的产品-市场匹配（PMF）

### 核心模型：跨越周期机制

**工具**：`蓝军 × 红军 × 战略框架`

| 角色 | 职责 |
|------|------|
| 蓝军 | 探索新机会，挑战现有业务 |
| 红军 | 优化现有业务，保住现金流 |
| 战略框架 | 决策机制，确保资源合理分配 |

### 输出格式

```
## 借势策略报告

**战略杠杆分析**：

| 要素 | 当前状态 | 评估 |
|------|----------|------|
| 支点 | 供应链优势 | ✓ 真实能力 |
| 杠杆 | 资本杠杆 | ⚠️ 使用不足 |
| 指北针 | 成为行业前三 | ✓ 清晰 |

**杠杆放大路径**：
融资 → 收购/整合 → 规模化

**利益相关人地图**：

| 相关人 | 影响力 | 利益 | 策略 |
|--------|--------|------|------|
| 核心供应商 | 高 | 高 | 重点管理，结成利益共同体 |
| 监管机构 | 高 | 低 | 保持合规，主动沟通 |
| 媒体 | 中 | 中 | 保持关系，危机预防 |

**闪电式扩张评估**：
- 产品-市场匹配：⚠️ 部分验证，需要扩大验证范围
- 建议：在PMF完全验证前，谨慎大规模扩张

**跨越周期准备**：
- 蓝军：❌ 尚未建立
- 红军：✓ 现有业务优化中
- 战略框架：⚠️ 需要完善

**核心建议**：
1. 融资加速，准备弹药
2. 建立蓝军机制，探索第二曲线
3. 完善战略决策框架，避免机会主义决策
```

## 模块六：用户深化

### 目的

让用户从流量变为共同体，建立持续增长的用户基础。

### 核心模型：用户深度四层

**公式**：`用户深化 = 流量 → 用户 → 会员 → 共同体`

| 层级 | 特征 | 核心指标 | 任务 |
|------|------|----------|------|
| 流量 | 一次性访问 | 曝光/点击 | 获取注意力 |
| 用户 | 注册/使用 | DAU/MAU | 激活价值 |
| 会员 | 付费/复购 | 付费率/复购率 | 建立关系 |
| 共同体 | 深度认同，传播 | NPS/UGC | 建立归属 |

**核心洞察**：四层不可跳级，需要层层深化。

### 核心模型：一横一纵框架

**纵轴**：`破局点 → 加杠杆 → 做分形`

| 阶段 | 任务 | 核心问题 |
|------|------|----------|
| 破局点 | 找到一个点做透 | 什么是你的破局点？ |
| 加杠杆 | 放大破局点优势 | 什么能杠杆化你的优势？ |
| 做分形 | 复制成功经验 | 如何在不同场景复制？ |

**横轴**：`设计模式 → 优化组织 → 借势 → 跨越`

| 维度 | 任务 |
|------|------|
| 模式 | 设计可持续的商业模式 |
| 组织 | 优化组织能力支撑增长 |
| 借势 | 借助外部资源放大 |
| 跨越 | 跨越非连续性，寻找第二曲线 |

### 输出格式

```
## 用户深化报告

**当前用户深度**：用户阶段（DAU 10万，会员转化率 3%）

**四层诊断**：

| 层级 | 状态 | 转化率 | 核心问题 |
|------|------|--------|----------|
| 流量 | ⚠️ 过度依赖投放 | - | 成本高，不可持续 |
| 用户 | ✓ 基础活跃 | 10% | 留存待优化 |
| 会员 | ⚠️ 转化不足 | 3% | 价值感知不够 |
| 共同体 | ❌ 尚未建立 | - | 需要长期运营 |

**核心问题**：用户价值感知不足，会员权益设计需要优化

**深化路径**：
```
流量（优化获取渠道，降低依赖）→ 用户（提升留存到15%）→ 会员（提升转化到8%）→ 共同体（建立UGC机制）
```

**一横一纵规划**：

纵轴规划：
1. 破局点：会员权益差异化
2. 加杠杆：口碑传播，老带新
3. 做分形：跨品类扩展
```

## 经验规则库

### 增长地图
1. 阶段不可跳，只能加速通过
2. 每个阶段有每个阶段的"正确"
3. 草莽阶段的核心任务是活下来
4. 腰部阶段是大多数企业的终局
5. 头部企业的核心能力是持续作出正确决定

### 机会识别
6. 破局点三要素缺一不可
7. 爆品=供应链创新×爆发品类×新流量
8. 品牌护城河三层，从易到难建立
9. 第二曲线要在第一曲线还在增长时探索
10. 穷举法是机会发现的基础

### 模式设计
11. 商业模式核心是"目标+要素+连接"
12. 增强回路是正反馈，调节回路是负反馈
13. 心理能量有尽头，靠模式不靠意志力
14. 用户增长飞轮=核心交付×增强回路×网络效应
15. S2B2C的核心是S赋能小B，小B服务C

### 组织诊断
16. 组织五阶段不可跨越，只能加速通过
17. 关系四阶段：理想→冲突→整合→协同
18. 权力三指标：信息流、利益链、资源调动
19. 透明、OKR、飞书是创新组织的三要素
20. 小团队用大组织方式会死

### 借势策略
21. 先找支点，再找杠杆，最后定方向
22. 杠杆放大需要支点稳定
23. 利益相关人地图帮助制定借势策略
24. 闪电式扩张前提是找到PMF
25. 蓝军红军机制帮助跨越周期

### 用户深化
26. 四层不可跳级，层层深化
27. 会员的核心是建立关系，不是卖卡
28. 共同体的核心是用户主动传播
29. 一横一纵是完整的增长规划框架

```python
def growth_diagnosis(situation):
    stage = classify_stage(situation)  # 企业定位模型

def score_breakthrough(opportunity):
    breadth = score_1_5("相对广谱")
    frequency = score_1_5("高频")
    control = score_1_5("体验可控")
    return breadth * frequency * control  # >60分才可能成为破局点

# 商业模式评估算法
def evaluate_model(business):
    goal = evaluate_goal(business["目标"])
    elements = evaluate_elements(business["要素"])
    connection = evaluate_connection(business["连接"])
    return (goal * elements * connection) ** (1/3)  # 几何平均

# 用户深度分类算法
def classify_user_depth(user_metrics):
    if user_metrics["NPS"] > 50 and user_metrics["UGC"] > 1000:
        return "共同体"
    elif user_metrics["付费率"] > 20:
        return "会员"
    elif user_metrics["DAU/MAU"] > 0.2:
        return "用户"
    else:
        return "流量"
```

## 陷阱反模式

| 陷阱 | 表现 | 纠正方法 |
|------|------|----------|
| 阶段错位 | 在草莽想头部的策略 | 先活下来，找到破局点 |
| 机会分散 | 同时做太多机会点 | 集中资源在一个破局点 |
| 破局点错选 | 选了但不具备三要素 | 重新评估，三要素缺一不可 |
| 模式模糊 | 说不清商业模式三要素 | 先画清楚"目标-要素-连接" |
| 回路缺失 | 只有增强没有调节 | 设计调节机制，防止过热 |
| 组织超前 | 小团队用大组织方式 | 尊重组织五阶段的时间窗 |
| 借势过度 | 依赖外部资源忽视内功 | 支点不稳，杠杆越大风险越大 |
| 闪电冒进 | PMF未验证就大规模扩张 | 先验证PMF，再闪电式扩张 |
| 周期幻觉 | 以为能跨越周期但实际不能 | 蓝军红军机制，避免盲目自信 |
| 用户浅薄 | 只关注流量忽视深化 | 从流量思维转向用户深度思维 |
| 跳级深化 | 想直接做共同体 | 四层不可跳级，层层深化 |
| 模式错配 | 连接器模式硬要做重 | 根据自身能力选择合适模式 |
