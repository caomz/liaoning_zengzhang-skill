# 草莽企业快速诊断示例

> 示例对话：新上线的SaaS产品用户增长缓慢

---

## 用户输入

```
我的B2B SaaS产品刚上线3个月，注册用户500人，付费用户只有12人，
月增长率不到10%，感觉很慢。我们团队5个人，技术出身，
产品功能做了很多但不知道哪个是核心。怎么办？
```

## 诊断过程

### Phase 0: 入口分流
→ 明确的增长问题 → **快速诊断路径 Phase 1A**

### Phase 1A: 问题分类

**归属模块**：增长地图 + 机会识别
**优先级**：① 阶段定位 → ② 破局点

---

## 诊断报告

### 阶段定位：草莽阶段

**企业定位模型**匹配：

| 阶段特征 | 匹配度 |
|---------|--------|
| 看到机会，快速冲 | ✓ |
| 团队小，创始人驱动 | ✓ |
| 没有明确竞争对手 | ✓ |
| 核心产品尚未验证 | ✓ |

**核心任务**：活下来。找到破局点。

### 模型调用：破局点公式

`破局点 = 相对广谱 × 高频 × 体验可控`

以你500个注册用户中付费最多的5个客户为样本：

| 评估维度 | 打分（1-5） | 分析 |
|---------|------------|------|
| 相对广谱 | 3 | B2B SaaS有细分市场，但覆盖面需验证 |
| 高频 | 2 | ⚠️ 用户使用频率低，说明核心功能不够刚需 |
| 体验可控 | 2 | ⚠️ 功能太多反而体验不可控 |
| **总分** | **12** | ❌ 远低于60分阈值 |

### 关键诊断

**核心问题**：破局点不清晰。

你做了很多功能，但没有一个功能是「相对广谱+高频+体验可控」的。典型的「机会分散」陷阱。

### 行动建议

1. **立即做**：找到付费的12个用户，逐个访谈——「你为什么付费？你最常用哪个功能？」
2. **这周做**：砍掉80%的功能，集中资源在付费用户最常用的那1个功能上
3. **本月做**：把这1个功能做到极致，用户体验可控且远超竞品

### ⚠️ 陷阱警示

| 陷阱 | 你的风险 | 纠正 |
|------|---------|------|
| 机会分散 | ⚠️ 高风险——功能太多 | 集中在一个破局点 |
| 阶段错位 | ⚠️ 中风险——想做太多 | 草莽阶段只做一件事：活下来 |

### 诚实边界

- 以上诊断基于你提供的有限信息，实际情况可能更复杂
- 破局点评分是近似值，需要结合用户访谈数据验证
- B2B SaaS 的增长节奏天然慢于 C 端产品，10%月增长在草莽阶段不一定是坏信号
