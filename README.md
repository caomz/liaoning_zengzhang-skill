## 效果示例

### 快速诊断模式

```
用户      ❯ 我的产品刚上线，用户增长很慢，怎么办？

增长顾问  ❯ 启动快速诊断——

            【阶段定位】：草莽阶段。核心任务是活下来，不是建立品牌。

            【模型调用】：破局点公式
            你的切入点具备「相对广谱 × 高频 × 体验可控」吗？

            关键诊断：大概率是破局点不清晰。你同时在做太多事。

            立即行动：
            1. 集中资源在一个破局点上，不要分散
            2. 用460件事工作坊穷举所有可能机会
            3. 对每个机会做破局点评分（三项乘积>60分才值得做）

            ⚠️ 陷阱警示：草莽阶段最容易掉的坑是「机会分散」
            ——同时做太多机会点，哪个都做不透。
```

### 深度咨询模式

```
用户      ❯ 我们在细分市场站稳了，想向头部冲击，但资源不够怎么办？

增长顾问  ❯ 启动借势策略深度分析——

            【阶段定位】：腰部企业 → 要向头部冲击

            【六大模块扫描摘要】
            ┌──────────┬──────┬────────────────────┐
            │ 模块      │ 状态  │ 关键发现            │
            ├──────────┼──────┼────────────────────┤
            │ 增长地图   │ ✓    │ 腰部阶段，定位清晰   │
            │ 机会识别   │ ⚠️   │ 第二曲线尚未启动     │
            │ 借势策略   │ ❌   │ 杠杆使用严重不足     │
            │ 组织诊断   │ ⚠️   │ 正从部落→村庄转型    │
            └──────────┴──────┴────────────────────┘
            核心瓶颈：借势策略

            【战略杠杆分析】
            核心问题：谁对你影响最大？你能给他什么长期利益？

            - 支点（你的核心能力）：细分市场的深度认知 ✓
            - 杠杆（资源放大器）：⚠️ 未找到，当前纯靠内生增长
            - 指北针（目标方向）：行业前三 ✓

            行动计划：
            P1（0-3月）：画利益相关人地图，找到关键盟友
            P2（3-6月）：让关键盟友与你共生，形成利益共同体
            P3（6-12月）：启动闪电式扩张（海军陆战队阶段）

            ⚠️ 支点不稳，杠杆越大风险越大。先确保核心能力不可替代。
```

### 特定企业分析

```
用户      ❯ 分析瑞幸咖啡的增长策略

增长顾问  ❯ [WebSearch获取真实数据...]

            启动企业增长诊断——

            【阶段定位】：头部企业（门店数超越星巴克中国）

            【增长飞轮分析】：
            核心交付：高性价比咖啡 + 便利性
            增强回路：门店密度↑ → 配送成本↓ → 价格竞争力↑ → 用户↑ → 门店密度↑
            网络效应：⚠️ 弱（咖啡消费个人化，社交属性低）

            【借势策略】：
            闪电式扩张阶段：军队阶段（规模化运营）
            战略杠杆：供应链规模效应 + 数字化运营能力

            诚实边界：以上分析基于公开信息，内部数据可能改变判断。
```

## 24个核心模型

### 增长地图
1. **企业定位模型** — 判断草莽/腰部/头部/顶级

### 机会识别
2. **穷举发散法** — 460件事工作坊
3. **破局点公式** — 相对广谱 × 高频 × 体验可控
4. **爆品公式** — 供应链创新 × 爆发品类 × 新流量
5. **品牌护城河** — 用户溢价/商场折扣/用户自拍
6. **第二曲线** — 跨越非连续性

### 模式设计
7. **商业模式三要素** — 目标 + 要素 + 连接
8. **增强回路** — 持续正反馈
9. **调节回路** — 心理能量有尽头
10. **用户增长飞轮** — 核心交付 × 增强回路 × 网络效应
11. **连接器模式** — 轻资产平台
12. **整合服务模式** — 重体验深连接
13. **流量平台模式** — 动态不平衡
14. **产业中台S2B2C** — 系统能力赋能小B

### 组织诊断
15. **组织五阶段** — 家庭→部落→村庄→城市→国家
16. **关系四阶段** — 理想→冲突→整合→协同
17. **权力三指标** — 信息流 × 利益链 × 资源调动
18. **字节跳动飞轮** — 透明 × OKR × 飞书

### 借势策略
19. **战略杠杆** — 支点 × 杠杆 × 指北针
20. **利益相关人地图** — 影响力 × 利益
21. **闪电式扩张** — 海盗→海军陆战队→军队→警察
22. **跨越周期机制** — 蓝军红军 × 战略框架

### 用户深化
23. **用户深度四层** — 流量→用户→会员→共同体
24. **一横一纵框架** — 纵：破局点→加杠杆→做分形

## 工作原理

```
用户输入 → 入口分流(Phase 0) → 信息采集(Phase 1) → 模型诊断(Phase 2) → 质量验证(Phase 3) → 诊断报告
```

**四条路径**：

| 路径 | 触发条件 | 时间 |
|------|---------|------|
| 🏃 快速诊断 | 明确的增长问题 | 2-5分钟 |
| 🔍 需求诊断 | 模糊困惑，先追问定位 | 5-10分钟 |
| 📊 深度咨询 | 全面战略规划 | 15-30分钟 |
| 🔬 研究先行 | 涉及具体企业/行业 | 先搜索再诊断 |

**关键设计**：
- **Phase 1.5 诊断确认检查点**：深度咨询时先展示六模块扫描摘要，用户确认再出建议
- **Phase 3 质量验证**：每个建议必须有模型支撑，标注陷阱和诚实边界
- **研究先行**：分析具体企业时必须先 WebSearch 获取真实数据

## 诚实边界

本技能明确标注以下局限：

- **不替代专业咨询** — 提供分析框架，不替代行业专家
- **模型有适用范围** — 偏向互联网行业，传统行业需调整
- **阶段判断是近似值** — 四阶段是简化框架
- **不预测未来** — 能诊断当前状态，不准确预测走向
